# Настройка приёма заявок

Фронтенд на GitHub Pages не хранит токены. Он отправляет заказ на отдельный
HTTPS endpoint, а тот дублирует заявку менеджеру.

## Рекомендуемая схема: Telegram + MAX

Готовый backend находится в
[`integrations/order-service`](integrations/order-service/README.md). Он:

- проверяет origin, размер и обязательные поля заявки;
- отправляет один и тот же текст в Telegram и MAX;
- проверяет фактический ответ каждого API;
- возвращает `success: true` только после доставки в оба канала;
- при повторе заявки досылает только в канал, который не сработал;
- хранит токены только в переменных окружения.

Для активации нужны:

1. `TELEGRAM_BOT_TOKEN` и числовой `TELEGRAM_CHAT_ID`.
2. Прошедший модерацию MAX-бот, `MAX_BOT_TOKEN` и `MAX_USER_ID` либо
   `MAX_CHAT_ID`.
3. Корневой сертификат Минцифры в секрете `MAX_CA_PEM`: актуальный API MAX
   `platform-api2.max.ru` требует добавить его в доверенные.
4. Публичный HTTPS-адрес развёрнутого Node-сервиса.

Номер телефона аккаунта MAX сам по себе **не является адресом Bot API**.
Владелец целевого аккаунта должен открыть MAX-бота и нажать **Начать**; после
события `bot_started` из `GET /updates` берётся числовой `user_id`/`chat_id`.

После деплоя задайте endpoint в `js/public-config.js`:

```js
window.BOAT_MAT_PUBLIC_CONFIG = {
  orderEndpoint: "https://orders.example.ru/orders",
  siteUrl: "https://kovrikvlodky.ru",
  telegramContact: "richywonderr"
};
```

Пока `orderEndpoint` пустой, сайт честно работает в ручном режиме:

- основная кнопка называется **«Подготовить заявку»**;
- текст копируется в буфер;
- открывается ручная кнопка Telegram `@richywonderr`;
- сайт не показывает, что заявка отправлена.

## Бесплатный резерв: только Telegram

Шаблон [`integrations/google-apps-script`](integrations/google-apps-script/README.md)
можно развернуть без отдельного сервера. Он подтверждает отправку в Telegram,
но не поддерживает MAX и не заменяет основную двухканальную схему.

## Что уходит в заявке

- номер и время заявки, версия расчёта;
- исходные и производственные размеры;
- площадь, периметр и количество частей;
- цена без раскрытия себестоимости и маржи;
- имя, телефон, комментарий и подтверждение размеров;
- текст заявки и SVG-чертёж.

## Безопасность

- Никогда не помещайте bot token, пароль или API secret в `index.html`,
  `public-config.js` или git.
- В `public-config.js` хранится только публичный URL endpoint.
- До публикации endpoint должен быть HTTPS; после публикации нужна тестовая
  заявка с подтверждением получения в обоих мессенджерах.
- В production задайте `STATE_FILE` на постоянном томе и поставьте endpoint за
  reverse proxy/WAF с внешним rate limit; `Origin` нельзя считать защитой от
  прямых запросов ботов.
