# Настройка приёма заявок

Сайт умеет отправлять заявки двумя способами.

## 1. Прямо сейчас — кнопка «Написать в Telegram» (без настройки)

На экране готовой заявки есть кнопка **«Написать в Telegram @richywonderr»**. Она открывает чат с менеджером, а текст заявки уже скопирован в буфер — клиент вставляет его (⌘/Ctrl+V) и отправляет. Работает без бэкенда и без секретов.

Изменить username менеджера: `js/public-config.js` → `telegramContact`.

## 2. Автоматическая отправка (по желанию)

Чтобы заявка уходила менеджеру автоматически (в Telegram и/или в Google-таблицу), разверните бесплатный бэкенд:

- Инструкция: [`integrations/google-apps-script/README.md`](integrations/google-apps-script/README.md)
- Код: [`integrations/google-apps-script/Code.gs`](integrations/google-apps-script/Code.gs)

После деплоя впишите URL в `js/public-config.js`:

```js
window.BOAT_MAT_PUBLIC_CONFIG = {
  orderEndpoint: "https://script.google.com/macros/s/…/exec",
  siteUrl: "https://kovrikvlodky.ru",
  telegramContact: "richywonderr"
};
```

Как только `orderEndpoint` задан:
- основная кнопка становится **«Отправить заявку»**;
- показываются статусы «Отправляем…» / «Заявка отправлена» / «Не удалось отправить»;
- ложного «успеха» без ответа сервера не бывает;
- при ошибке клиент всегда может отправить заявку через кнопку Telegram.

## Что уходит в заявке

- номер, дата, версия расчёта;
- исходные и производственные размеры;
- площадь, периметр, количество частей;
- цена (без раскрытия себестоимости/маржи);
- контакты и согласие;
- текст заявки и SVG-чертёж.

## Безопасность

- Во фронтенде и репозитории **нет** токенов и паролей — только публичный URL endpoint.
- Токен бота и chat_id хранятся в Script Properties Apps Script.
- Контактные данные не хранятся в публичном репозитории.
